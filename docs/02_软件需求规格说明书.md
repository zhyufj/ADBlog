# 软件需求规格说明书

## 1. 引言

### 1.1 项目背景

在日常测试工作中，组里的测试人员习惯使用手机上的文件系统导出日志给开发分析。这种方式不仅操作繁琐，而且不同品牌手机导出日志的方法各不相同。ADBlog 就是为解决这个问题开发的——测试人员只需连接手机、双击运行，就能实时查看应用日志，不用记 ADB 命令，也不受手机品牌影响。

### 1.2 术语定义

| 术语 | 说明 |
|------|------|
| ADB | Android Debug Bridge，Android 调试桥，用于电脑与 Android 设备通信 |
| Logcat | Android 日志系统，实时输出系统和应用日志 |
| 包名（Package Name） | Android 应用的唯一标识符，如 `com.example.app` |
| USB 调试 | Android 设备的开发者选项，允许通过 ADB 进行调试操作 |
| 日志级别 | Android 日志的严重程度分级：V(Verbose)、D(Debug)、I(Info)、W(Warning)、E(Error)、F(Fatal) |

### 1.3 参考资料

- Android 官方 ADB 文档
- Go 语言标准库文档
- 项目 Git 仓库：`https://github.com/zhyufj/ADBlog`

## 2. 系统概述

### 2.1 系统目标

开发一款 Windows 命令行工具，通过 ADB 协议连接 Android 设备，实现日志的实时抓取、过滤和彩色显示，降低测试人员获取日志的门槛。

### 2.2 用户角色

| 角色 | 描述 |
|------|------|
| 测试人员 | 主要用户，使用该工具查看被测应用的运行日志 |
| 开发人员 | 辅助用户，通过该工具快速定位问题 |

### 2.3 运行环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| ADB 环境 | 程序自带 adb.exe，无需单独安装 |
| 手机端 | Android 设备，已开启 USB 调试并授权 |
| 其他依赖 | 无（单文件可执行程序） |

## 3. 功能需求

### 3.1 功能模块总览

```
ADBlog
├── 模块1：设备检测与管理
│   ├── FR-01 单设备自动识别
│   ├── FR-02 多设备手动选择
│   └── FR-03 设备状态过滤（仅识别 device 状态）
├── 模块2：包名输入与过滤
│   ├── FR-04 输入"0"自动获取当前前台应用包名
│   ├── FR-05 输入"9"列出所有已安装应用包名
│   ├── FR-06 直接输入指定包名
│   └── FR-07 留空不设置包名过滤
├── 模块3：日志级别选择
│   ├── FR-08 六级日志级别选择（F/E/W/I/D/V）
│   └── FR-09 默认级别设置（E 级别）
├── 模块4：日志抓取与显示
│   ├── FR-10 实时日志抓取（调用 adb logcat）
│   ├── FR-11 日志级别过滤
│   ├── FR-12 包名关键字过滤
│   └── FR-13 彩色高亮输出（按级别分色）
└── 模块5：辅助功能
    ├── FR-14 logcat 缓冲区自动清空
    └── FR-15 Ctrl+C 安全退出
```

### 3.2 功能需求详述

#### FR-01 单设备自动识别

- **描述：** 当仅有一台 Android 设备通过 USB 连接且状态为 `device` 时，程序自动选择该设备。
- **输入：** 无（程序自动执行 `adb devices`）
- **输出：** 显示 "检测到设备: `<设备序列号>`"，自动进入下一步。
- **优先级：** P0（核心功能）

#### FR-02 多设备手动选择

- **描述：** 当有多台 Android 设备同时连接时，列出所有设备供用户选择。
- **输入：** 用户输入设备序号（从 1 开始的整数）
- **输出：** 选择对应设备进入下一步；输入无效序号时默认选择第一台设备。
- **优先级：** P0

#### FR-03 设备状态过滤

- **描述：** 仅将状态为 `device` 的设备列为可用设备；状态为 `unauthorized`、`offline` 等的设备不显示在列表中。
- **异常处理：** 无可用设备时，显示 "未检测到设备" 后程序退出。
- **优先级：** P0

#### FR-04 自动获取当前应用包名（输入 0）

- **描述：** 用户输入 `0`，程序通过 `adb shell dumpsys window` 获取当前前台应用的包名。
- **输出：** 成功时显示 "检测到当前应用: `<包名>`"；失败时显示 "未检测到当前应用，将显示全部日志"。
- **优先级：** P0

#### FR-05 列出所有应用包名（输入 9）

- **描述：** 用户输入 `9`，程序通过 `adb shell pm list packages` 列出设备所有已安装应用的包名。
- **输出：** 每行一个包名（去除 `package:` 前缀），列表输出完毕后提示用户输入要抓日志的包名。
- **优先级：** P0

#### FR-06 直接输入指定包名

- **描述：** 用户直接输入目标应用的完整包名（如 `com.example.app`），程序仅显示包含该包名字符串的日志行。
- **匹配规则：** 使用字符串包含匹配（`strings.Contains`），区分大小写。
- **优先级：** P0

#### FR-07 留空不设置包名过滤

- **描述：** 用户不输入任何内容直接回车，程序不进行包名过滤，显示所有应用的日志。
- **优先级：** P0

#### FR-08 六级日志级别选择

- **描述：** 用户可选择 1~6 对应 F/E/W/I/D/V 六个日志级别。
- **过滤规则：** 仅显示级别值 ≥ 所选级别的日志行。级别映射如下：

| 选项 | 级别 | 含义 | 权重值 | 显示范围 |
|------|------|------|--------|----------|
| 1 | F | Fatal（严重错误） | 5 | 仅 F |
| 2 | E | Error（错误） | 4 | E + F |
| 3 | W | Warning（警告） | 3 | W + E + F |
| 4 | I | Info（信息） | 2 | I + W + E + F |
| 5 | D | Debug（调试） | 1 | D + I + W + E + F |
| 6 | V | Verbose（详细） | 0 | 全部级别 |

- **优先级：** P0

#### FR-09 默认级别设置

- **描述：** 用户输入无效选项或直接回车时，默认选择 E（Error）级别。
- **优先级：** P0

#### FR-10 实时日志抓取

- **描述：** 调用 `adb -s <device> logcat` 持续读取设备日志流。
- **前置动作：** 先执行 `adb -s <device> logcat -c` 清空历史缓冲区。
- **输出：** 日志实时逐行显示在终端中。
- **优先级：** P0

#### FR-11 日志级别过滤

- **描述：** 从每行日志的前 50 个字符中提取级别标记，按用户设置的级别阈值进行过滤。
- **级别提取规则：** 逐字符扫描前 50 字符，找到第一个满足 "前导为空格或行首 + [VDIWEF] + 后随 / 或空格" 的位置，返回对应字符作为日志级别。此方法避免了消息体中包含级别关键词时的误判。
- **优先级：** P0

#### FR-12 包名关键字过滤

- **描述：** 若设置了包名过滤，仅显示日志内容中包含该包名字符串的日志行。
- **优先级：** P0

#### FR-13 彩色高亮输出

- **描述：** 不同级别的日志以不同颜色显示，便于快速识别：

| 级别 | 颜色效果 | ANSI 转义码 |
|------|----------|-------------|
| F | 红底白字 | `\033[41;37m` |
| E | 红色文字 | `\033[31m` |
| W | 黄色文字 | `\033[33m` |
| I | 绿色文字 | `\033[32m` |
| D | 青色文字 | `\033[36m` |
| V | 默认颜色 | 无 |

- **优先级：** P0

#### FR-14 logcat 缓冲区自动清空

- **描述：** 启动日志抓取前，自动执行 `logcat -c` 清空设备日志缓冲区，确保只显示程序启动后的新日志。
- **优先级：** P1

#### FR-15 Ctrl+C 安全退出

- **描述：** 用户在终端按 Ctrl+C 组合键，程序终止 logcat 进程并正常退出。
- **优先级：** P0

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 日志单行缓冲区 | 1MB（兼容超长堆栈信息） |

程序为轻量级命令行工具，正常情况下 CPU 和内存占用很低，无特殊性能指标要求。

### 4.2 可用性需求

- 双击 EXE 即可运行，无需命令行参数配置
- 交互式菜单引导，减少用户记忆负担
- 错误提示使用中文，便于理解

### 4.3 可靠性需求

- ADB 连接异常时在终端中显示错误信息（通过命令行启动可见），而非静默退出
- 设备断开后程序不崩溃
- 长时间运行内存占用稳定

### 4.4 可维护性需求

- 源代码使用 Go 语言编写，单一文件结构清晰
- 核心函数职责单一，便于修改和扩展
- 提供单元测试覆盖核心逻辑

### 4.5 兼容性需求

- 兼容 Windows 10 / Windows 11
- 兼容主流 Android 设备（需开启 USB 调试）
- 兼容不同品牌的 Android 设备

---

## 5. 约束条件

1. 必须依赖 ADB 协议，设备需开启 USB 调试
2. 仅支持 USB 有线连接（无线 ADB 需额外配置）
3. 仅支持 Windows 平台（ADB 路径硬编码为 `.\adb.exe`）
4. Go 语言版本 ≥ 1.25

| 编号 | 验收项 | 验收标准 |
