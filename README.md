ADBlog
一个简单易用的Windows命令行工具，通过ADB实时查看Android设备日志，支持日志级别过滤和彩色高亮显示。

项目背景
在日常测试工作中，我发现组里的测试人员习惯使用手机上的文件系统来导出日志给开发人员分析。这种方式不仅操作繁琐，而且不同品牌的手机导出日志的方法各不相同，给测试工作带来了很大的不便。为了解决这个问题，我开发了这个基于ADB的日志查看工具，让测试人员只需要连接手机、双击运行，就能实时查看手机软件的运行日志，无需记忆复杂的ADB命令，也不用关心不同手机的日志导出方式。

功能特点
自动检测设备：自动识别已连接的Android设备，支持多设备选择

彩色日志输出：不同日志级别显示不同颜色（F红底白字、E红色、W黄色、I绿色、D青色）

日志级别过滤：支持V/D/I/W/E/F各级别日志过滤

包名过滤：可按应用包名过滤日志，支持直接输入包名、输入0自动获取当前应用、输入9列出所有应用

即插即用：连接手机后双击运行，无需配置

使用方法
Android设备开启USB调试并连接到电脑

双击运行ADBlog.exe

根据提示输入包名（0=当前APP，9=列出所有APP，可留空）

选择日志级别（默认2=E级）

实时查看彩色日志输出

环境要求
Windows操作系统
ADB已配置（程序自带adb.exe，放在同一目录下即可）
Android设备已开启USB调试

ADBlog/
├── main.go          # 主程序
├── adb.exe          # ADB可执行文件
├── README.md        # 说明文档
└── ADBlog.exe    # 编译后的可执行文件

注意：
如果手机没有打开USB调试或其他原因导致ADB连接失败，程序会直接闪退，可以通过“在终端中打开”的方式，输入./ADBlog.exe的方式启动程序查看具体错误信息
