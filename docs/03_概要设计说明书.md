# 软件概要设计说明书

## 1. 设计原则

- **单一职责：** 每个函数只做一件事
- **简单优先：** 不引入多余的抽象层，保持代码直白
- **即拆即用：** 编译为单文件 EXE，自带 adb.exe
- **容错：** 异常情况用中文提示，不闪退

---

## 2. 系统架构

### 2.1 总体架构

系统采用 CLI（命令行界面）架构，内部按职责分为交互控制层、业务逻辑层和外部调用层，全部编译在单个可执行文件中。

```
┌─────────────────────────────────────────┐
│                 用  户                    │
│            （Windows 终端）               │
└──────────────┬──────────────────────────┘
               │ 交互式输入（键盘）
               ▼
┌─────────────────────────────────────────┐
│              ADBlog.exe                  │
│  ┌──────────────────────────────────┐   │
│  │        交互控制层 (main)           │   │
│  │  流程编排 · 用户交互 · 输入解析    │   │
│  └──────────────┬───────────────────┘   │
│                 │                        │
│  ┌──────────────▼───────────────────┐   │
│  │        业务逻辑层                  │   │
│  │  ┌──────────┐  ┌──────────────┐  │   │
│  │  │ 设备管理  │  │ 日志过滤引擎  │  │   │
│  │  │·select   │  │·extractLevel │  │   │
│  │  │ Device   │  │·processLine  │  │   │
│  │  │·getPkg   │  │·printColored │  │   │
│  │  │·listPkg  │  │·chooseLevel  │  │   │
│  │  └─────┬────┘  └──────┬───────┘  │   │
│  └────────┼──────────────┼──────────┘   │
│           │              │               │
│  ┌────────▼──────────────▼──────────┐   │
│  │        外部调用层                  │   │
│  │     exec.Command(adbPath, ...)    │   │
│  └──────────────┬───────────────────┘   │
└─────────────────┼───────────────────────┘
                  │ adb protocol (USB)
                  ▼
┌─────────────────────────────────────────┐
│          Android 设备                     │
│    ┌──────────┐    ┌────────────────┐   │
│    │ adb daemon│   │  logcat buffer │   │
│    └──────────┘    └────────────────┘   │

```

### 2.2 技术选型

| 技术项 | 选择 | 理由 |
|--------|------|------|
| 编程语言 | Go 1.25 | 编译为单文件可执行程序，无需运行时依赖 |
| 设备通信 | ADB (adb.exe) | Android 标准调试协议，自带 adb.exe |
| 终端输出 | ANSI 转义码 | Windows 10+ 终端原生支持，无需第三方库 |
| 用户交互 | bufio.Reader + os.Stdin | Go 标准库，简单可靠 |
| 外部命令调用 | os/exec | Go 标准库，同步执行并捕获输出 |

## 3. 模块划分

### 3.1 模块结构

```
main.go (单一源文件，按职责划分函数组)
│
├── 入口与流程控制
│   └── main()                          # 主流程编排
│
├── 设备管理模块
│   ├── selectDevice()                  # 设备检测与选择
│   ├── getCurrentPackage()             # 获取当前前台应用
│   └── listPackages()                  # 列出所有应用包名
│
├── 用户输入模块
│   └── chooseLevel()                   # 日志级别选择与校验
│
├── 日志处理模块
│   ├── startLogcat()                   # 启动 logcat 并读取
│   ├── processLine()                   # 单行日志过滤分发
│   ├── extractLevel()                  # 日志级别提取
│   └── printColored()                  # 彩色输出
│
└── 全局配置
    ├── adbPath                         # ADB 可执行文件路径
    └── levelOrder                      # 日志级别权重映射表
```

### 3.2 模块职责

#### 模块 1：设备管理模块

| 函数 | 职责 | 输入 | 输出 |
|------|------|------|------|
| `selectDevice` | 执行 `adb devices`，解析设备列表，处理用户选择 | bufio.Reader | 设备序列号 string |
| `getCurrentPackage` | 执行 `adb shell dumpsys window`，解析前台应用包名 | 设备序列号 | 包名 string |
| `listPackages` | 执行 `adb shell pm list packages`，输出所有包名 | 设备序列号 | 无（直接输出） |

#### 模块 2：用户输入模块

| 函数 | 职责 | 输入 | 输出 |
|------|------|------|------|
| `chooseLevel` | 显示级别菜单，读取并校验用户输入 | bufio.Reader | 级别字母 string |

#### 模块 3：日志处理模块

| 函数 | 职责 | 输入 | 输出 |
|------|------|------|------|
| `startLogcat` | 清空缓冲区后启动 `adb logcat`，逐行读取 | 设备、包名、级别 | 无（持续流式输出） |
| `processLine` | 对单行日志进行包名和级别双重过滤 | 日志行、包名、最小级别 | 无（调用 printColored） |
| `extractLevel` | 从日志行中提取级别标记 | 日志行 | 级别字母 string |
| `printColored` | 按级别输出带 ANSI 颜色码的文本 | 日志行、级别 | 无（输出到 stdout） |

## 4. 核心数据流

### 4.1 主流程数据流

```
用户启动程序
    │
    ▼
┌─────────────┐    adb devices     ┌──────────┐
│ selectDevice │ ──────────────────▶│ ADB 守护进程│
└──────┬──────┘                    └────┬─────┘
       │ 设备序列号                     │
       ▼                                │
┌──────────────┐                        │
│  包名输入判断  │                        │
│ 0/9/自定义/空 │                        │
└──────┬───────┘                        │
       │                                │
       ├─ 0 → getCurrentPackage() ──────┤ adb shell dumpsys window
       ├─ 9 → listPackages() ───────────┤ adb shell pm list packages
       ├─ 自定义 → 直接使用               │
       └─ 空 → 跳过过滤                  │
       │                                │
       ▼                                │
┌──────────────┐                        │
│  chooseLevel │                        │
└──────┬───────┘                        │
       │ 级别字母                        │
       ▼                                │
┌──────────────┐    adb logcat -c      │
│ startLogcat  │ ───────────────────────┤
│              │    adb logcat          │
│   ┌──────────┴◄───────────────────────┘
│   │ 逐行读取
│   ▼
│ processLine(line, pkg, level)
│   ├── 包名过滤
│   ├── 级别提取 (extractLevel)
│   ├── 级别过滤 (levelOrder 比较)
│   └── printColored(line, level) → stdout (ANSI 彩色)
│
└── Ctrl+C → 退出
```

### 4.2 日志级别过滤数据流

```
原始日志行
    │
    ▼
┌────────────────────┐
│ extractLevel(line)  │  取前50字符逐字符扫描
│                     │  匹配规则: (行首|空格)+[VDIWEF]+(/|空格)
└────────┬───────────┘
         │ 级别字母 或 ""
         ▼
    level == "" ? ──── yes ──▶ 过滤（不输出）
         │
        no
         │
         ▼
  levelOrder[level]          比较过滤
       <                ──── yes ──▶ 过滤（不输出）
  levelOrder[minLevel]
         │
        no
         │
         ▼
┌────────────────────┐
│ printColored()     │  ANSI 彩色输出到终端
└────────────────────┘
```

## 5. 接口设计

### 5.1 外部接口

#### ADB 命令行接口

| 命令 | 用途 | 调用位置 |
|------|------|----------|
| `adb devices` | 获取已连接设备列表 | `selectDevice()` |
| `adb -s <dev> shell dumpsys window` | 获取当前窗口信息 | `getCurrentPackage()` |
| `adb -s <dev> shell pm list packages` | 获取已安装应用列表 | `listPackages()` |
| `adb -s <dev> logcat -c` | 清空日志缓冲区 | `startLogcat()` |
| `adb -s <dev> logcat` | 持续输出日志流 | `startLogcat()` |

### 5.2 内部函数接口

所有函数均为包内函数（小写开头），无外部包依赖（除 Go 标准库）。

### 5.3 用户接口

程序通过终端标准输入/输出与用户交互：

- **输入设备：** 键盘（stdin）
- **输出设备：** 终端屏幕（stdout）
- **交互模式：** 行式问答（逐行提示-输入）

## 6. 数据结构

### 6.1 全局变量

```go
// ADB 可执行文件路径（Windows 相对路径）
var adbPath = ".\\adb.exe"

// 日志级别权重映射表（权重值越大级别越高）
var levelOrder = map[string]int{
    "V": 0,  // Verbose
    "D": 1,  // Debug
    "I": 2,  // Info
    "W": 3,  // Warning
    "E": 4,  // Error
    "F": 5,  // Fatal
}
```

## 7. 关键设计决策

| 决策点 | 决策 | 理由 |
|--------|------|------|
| 单文件 vs 多模块 | 单文件 | 项目规模小，单文件降低维护成本 |
| ADB 路径 | 相对路径 `.\adb.exe` | 用户直接解压到同一目录即可使用 |
| 级别提取范围 | 仅检查前 50 字符 | 级别标记总是在日志行头部，50字符足够且避免长消息误判 |
| 级别提取方式 | 逐字符扫描 | 避免 `strings.Contains` 将消息体中的 "I/xxx" 误判为 I 级别 |
| 包名匹配方式 | 子串包含匹配 | 简单高效，覆盖 Tag 包含包名的场景 |
| ANSI 颜色 | 内联转义码 | 无需第三方库，Windows 10+ 终端原生支持 |
