# 项目开发计划

## 1. 项目背景

日常测试中，组里测试人员习惯从手机文件系统导出日志给开发，不同品牌手机方法各异、操作繁琐。ADBlog 的目标：连接手机、双击运行，就能实时查看应用日志。

### 项目概况

| 属性 | 内容 |
|------|------|
| 项目名称 | ADBlog — Android Debug Bridge Log Viewer |
| 项目类型 | 桌面命令行工具 |
| 开发语言 | Go 1.25 |
| 目标平台 | Windows 10/11 |
| 开发模式 | 个人独立开发 |
| 版本号 | v1.0 |

---

## 2. 项目目标与范围

### 2.1 项目目标

开发一款 Windows 命令行工具，通过 ADB 连接 Android 设备，实现：
- 自动检测已连接设备
- 实时抓取并显示 Android 日志
- 支持按应用包名和日志级别双重过滤
- 不同级别日志以不同颜色高亮显示

### 2.2 功能范围

| 功能模块 | 功能点 | 优先级 |
|----------|--------|--------|
| 设备管理 | ADB 设备检测、多设备选择、设备状态过滤 | P0 |
| 包名输入 | 输入0自动获取当前应用、输入9列出所有应用、直接输入包名、留空不过滤 | P0 |
| 级别选择 | 6级日志级别选择（F/E/W/I/D/V）、默认E级别 | P0 |
| 日志抓取 | 实时 logcat 读取、缓冲区清空 | P0 |
| 日志过滤 | 包名关键字匹配、级别阈值比较 | P0 |
| 彩色输出 | 6种 ANSI 颜色按级别区分 | P0 |

### 2.3 不包含的功能

- 无线 ADB (WiFi ADB) 连接支持
- Mac/Linux 平台支持
- 日志导出到文件功能
- GUI 图形界面
- 正则表达式高级过滤

## 5. 风险管理

| 风险编号 | 风险描述 | 影响 | 概率 | 应对措施 |
|----------|----------|------|------|----------|
| R1 | 不同品牌手机 ADB 兼容性问题 | 部分设备无法正常读取日志 | 中 | 准备多品牌测试机，使用标准 ADB 协议 |
| R2 | Windows 终端 ANSI 颜色不支持 | 彩色输出失效（功能降级） | 低 | Windows 10+ 原生支持，旧版本提示用户升级 |
| R3 | logcat 输出格式变化 | 级别提取失败 | 低 | 支持 brief 和 threadtime 两种主流格式 |
| R4 | Go 版本兼容问题 | 编译失败 | 低 | 固定 Go 1.25，go.mod 锁定版本 |
