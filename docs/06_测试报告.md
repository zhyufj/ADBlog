### 3.2 关键测试用例详解

#### 3.2.1 extractLevel 消息体误判修复验证

这是本次测试的重中之重——验证 `extractLevel` 函数修复后，消息体中的 "I/"、"D/" 等文本不会被误判为日志级别。

| 子用例 | 输入 | 预期 | 验证点 |
|--------|------|------|--------|
| E级别消息含I/ | `E/MyApp(12345): Error reading I/Stream data` | `E` | 行首 E 被正确匹配，消息体 I/ 被忽略 |
| W级别消息含E/ | `W/MyApp(12345): caused by E/Network error` | `W` | 行首 W 被正确匹配，消息体 E/ 被忽略 |
| threadtime-E级别消息含I/ | `06-22 ... E MyApp: Error reading I/Stream` | `E` | threadtime 格式中 E 被正确匹配 |

**修复前后对比：**

```
修复前 (strings.Contains 版本):
  输入: "E/MyApp(12345): Error reading I/Stream data"
  扫描: Contains(" I/") → true → 返回 "I"  ❌ 错误！
  
修复后 (逐字符扫描版本):
  输入: "E/MyApp(12345): Error reading I/Stream data"
  扫描: i=0, ch='E', 行首✓, 后继='/'✓ → 返回 "E"  ✓ 正确！
```

## 5. 缺陷统计

### 5.1 已修复缺陷

| 编号 | 严重程度 | 描述 | 修复状态 |
|------|----------|------|----------|
| BUG-001 | P0-严重 | extractLevel 消息体级别误判（E级日志含 "I/Stream" 被误判为I级） | ✅ 已修复 |
| BUG-002 | P2-体验 | 多设备选择输入无效序号时无提示 | ✅ 已修复 |
| BUG-003 | P2-体验 | chooseLevel 中 "2" 走 default 分支而非显式 case，代码意图不清晰 | ✅ 已添加显式 case "2" |

### 5.2 已知限制

| 编号 | 描述 | 影响 | 计划 |
|------|------|------|------|
| LIM-001 | 包名匹配为大小写敏感（`strings.Contains`），输入 "Com.Example.App" 无法匹配 "com.example.app" | 用户需输入正确大小写 | 后续可加 `strings.ToLower` |
| LIM-002 | extractLevel 仅检查前 50 字符，极端情况下 tag 名超长的日志行可能漏判 | 极少发生 | 可配置化 |
| LIM-003 | 仅支持 Windows 平台 | Mac/Linux 需手动修改 adbPath | 后续可支持跨平台 |

### 6.3 建议

1. **持续集成：** 建议将 `go test` 加入 CI 流水线，每次提交自动运行单元测试

3. **Mock 优化：** 将 `exec.Command` 抽象为接口，实现更完整的单元测试覆盖（当前 ADB 相关函数依赖实际设备）
4. **跨平台支持：** 当需要支持 Mac/Linux 时，将 `adbPath` 改为平台自适应
