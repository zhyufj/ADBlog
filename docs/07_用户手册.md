# 用户手册

## 1. 产品简介

ADBlog 是一款 Windows 命令行工具，通过 ADB 协议连接 Android 设备，实时查看设备日志。支持按应用包名和日志级别双重过滤，不同级别日志以不同颜色高亮显示。

**适用人群：** 软件测试人员、Android 开发人员。

**核心价值：** 无需记忆 ADB 命令，无需关心手机品牌差异，即插即用。

---

## 2. 安装与准备

### 2.1 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 或 Windows 11 |
| 手机系统 | Android（需支持 USB 调试） |
| 连接方式 | USB 数据线 |
| 其他 | 无需安装任何额外软件 |

### 2.2 安装步骤

ADBlog 为绿色免安装软件：

1. 将程序包解压到任意目录
2. 确保 `ADBlog.exe` 和 `adb.exe` 在同一目录下
3. 完成

### 2.3 手机端设置

1. 打开手机 **「设置」** → **「关于手机」**
2. 连续点击 **「版本号」** 7 次，激活 **「开发者选项」**
3. 进入 **「开发者选项」**，开启 **「USB 调试」**
4. 使用 USB 数据线连接手机到电脑
5. 手机上弹出 **「允许 USB 调试」** 弹窗时，勾选 **「一律允许」**，点击 **「确定」**

> ⚠️ 如果未弹出授权弹窗，请检查：USB 线是否支持数据传输（非仅充电线）、手机 USB 连接模式是否为「传输文件」。

## 3. 使用指南

### 3.1 启动程序

双击 `ADBlog.exe` 即可启动。

> 💡 如果双击后窗口一闪而过，请在程序所在目录空白处按住 Shift + 右键 → **「在此处打开 PowerShell 窗口」**，输入 `.\ADBlog.exe` 回车，即可看到具体错误信息。

### 3.2 操作流程

#### 步骤 1：设备选择

```
检测到设备: ABC123456789
```

- **单设备：** 自动选择，无需操作
- **多设备：** 显示设备列表，输入序号选择：

```
检测到多个设备:
1. ABC123456789
2. DEF987654321
请选择设备: _
```

- **无设备：** 显示"未检测到设备"后退出，请检查手机 USB 调试是否开启

#### 步骤 2：包名输入

```
请输入包名 (0=当前APP, 9=列出所有APP, 可留空): _
```

| 输入 | 效果 |
|------|------|
| `0` | 自动获取当前前台运行的 APP 包名 |
| `9` | 列出手机中所有已安装 APP 的包名，然后再次输入 |
| `com.xxx.xxx` | 直接输入包名，仅显示该 APP 的日志 |
| 直接回车 | 不设置包名过滤，显示所有 APP 的日志 |

**示例：**

```
# 输入 0
请输入包名: 0
检测到当前应用: com.tencent.mm

# 输入 9
请输入包名: 9
正在获取所有APP包名...
com.android.settings
com.android.chrome
com.tencent.mm
...
请输入需要抓日志的包名: com.tencent.mm
```

#### 步骤 3：日志级别选择

```
选择日志级别:
1. F - 严重
2. E - 错误
3. W - 警告
4. I - 信息
5. D - 调试
6. V - 全部
选择 (默认2): _
```

| 选项 | 级别 | 说明 | 推荐场景 |
|:----:|------|------|----------|
| 1 | F | 仅显示致命错误 | 排查崩溃问题 |
| 2 | E | 显示错误及以上 | **日常测试（推荐默认）** |
| 3 | W | 显示警告及以上 | 关注潜在问题 |
| 4 | I | 显示信息及以上 | 查看应用运行流程 |
| 5 | D | 显示调试及以上 | 详细调试分析 |
| 6 | V | 显示全部日志 | 最详细，日志量大 |

> 直接回车默认选择 **2（E 级别）**。

#### 步骤 4：查看日志

```
开始抓取日志 (Ctrl+C退出)
设备: ABC123456789
最低日志级别: E
包名过滤: com.tencent.mm
-------------------------------------
```

日志实时滚动输出，不同级别显示不同颜色。

#### 步骤 5：退出

按下 **Ctrl + C** 组合键即可退出程序。

## 4. 日志颜色说明

| 颜色 | 级别 | 含义 |
|------|------|------|
| 🔴 红底白字 | F (Fatal) | 致命错误，程序崩溃 |
| 🔴 红色 | E (Error) | 错误，功能异常 |
| 🟡 黄色 | W (Warning) | 警告，潜在问题 |
| 🟢 绿色 | I (Info) | 信息，正常运行记录 |
| 🔵 青色 | D (Debug) | 调试，开发调试信息 |
| ⚪ 默认色 | V (Verbose) | 详细，最低级别输出 |

## 5. 常见问题

### Q1：双击后窗口闪退怎么办？

在程序目录空白处按住 **Shift + 右键** → **「在此处打开 PowerShell 窗口」**，输入 `.\ADBlog.exe` 查看错误提示。常见原因：
- 手机未开启 USB 调试
- USB 线不支持数据传输
- 手机未授权电脑的 USB 调试请求

### Q2：检测不到设备？

1. 确认手机 USB 调试已开启
2. 重新插拔 USB 线，观察手机是否有授权弹窗
3. 在 PowerShell 中执行 `.\adb.exe devices`，查看设备状态
4. 如果状态为 `unauthorized`，解锁手机屏幕查看授权弹窗
5. 如果状态为 `offline`，重启手机的 USB 调试开关

### Q3：输入 0 提示"未检测到当前应用"？

手机可能处于锁屏状态或桌面，解锁手机并打开目标应用后重新运行程序。

### Q4：日志太多太快看不清？

选择更高的日志级别（如 E 或 F），减少不重要的日志输出。

### Q5：怎么只查看某个 App 的日志？

在包名输入步骤输入该 App 的包名。如果不知道包名，输入 9 查看所有已安装应用的包名列表。

### Q6：程序支持无线 ADB 吗？

v1.0 仅支持 USB 有线连接。无线 ADB 需要额外配置，后续版本考虑支持。

## 6. 注意事项

1. 首次连接手机时，务必在手机上授权「允许 USB 调试」
2. 程序所在目录需要保持 `adb.exe` 与 `ADBlog.exe` 在同一目录
3. 部分安全软件可能拦截 ADB 通信，如遇连接问题可尝试临时关闭防火墙
4. 日志抓取期间请勿断开 USB 连接，否则需要重启程序
5. 长时间抓取日志会产生大量输出，建议按需选择日志级别
